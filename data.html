Welcome to the AusPass data portal


  • How to get AusPass data: AusPass data can be accessed with the WebDC3 graphical explorer (below) or with the (preferred) FDSN seismic data standard. SeedLink access (port 18000) is also available, but please note that Arclink is no longer supported.
  • How to acknowledge AusPass: When using AusPass data, please acknowledge AusPass and reference each array separately. Complete references are given on array webpages, accessible on our homepage and resources. References and DOI information is also listed in the station XML metadata for each network (these tend to be more up to date). An example of appropriate acknowledgments is given in our data policy.
  • Do you need help? Visit our tutorials to explore the functionnality of our graphical interface (basic and advanced tutorials). You can also learn how to access AusPass data with a standard FDSN request (generic request and via an ObsPy tutorial). If you still need help, please, contact us at auspass@anu.edu.au.
  • We appreciate your feedback! To help us improve AusPass, please tell us how you used our data. References to publications using AusPass are warmly welcomed.

  • We are happy to announce the webdc3 data portal is back online! However we still encourage users to explore these alternative options:
    1) Our FDSN web server. You can build waveform data or station/response metadata queries easily (by hand or by script!) and download them in your browser or via wget/curl et al.
    2) PyWeed, a standalone GUI program from IRIS (win/osx/linux) demo video. NOTE: the version linked here has been hacked to be compatible with AusPass; later versions should hopefully include this AusPass compatibility by default.
    3) An increasing variety of FDSN-compatible software and codes (e.g. ObsPy, irisFetch-matlab, or even a GIS plugin QQuake)
